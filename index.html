Welcome to Trace Tables

A trace table is a reliable technique used to dry-run (test) algorithms manually. It helps ensure no logical errors occur while the program processes data.

  • OCR J277 Compliant Code: All code generated on this tool explicitly matches the OCR Exam Reference Language to prepare you for your real exam format.
  • OCR J277 Marking Rule: In OCR exams, you only add a new row to your trace table when a variable changes/is assigned, or if there is an output.
  • Ignore the Fluff: Do not include lines for condition checks (like while x > 5 or if x == 2) or loop markers (like next i or endwhile) unless a variable state is updated on that exact line.
  • Variable Tracking: Only write down a variable's value when it changes. If it stays the same, leave the cell blank.
